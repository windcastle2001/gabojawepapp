## 1. Executive Summary

### 1.1 제품 한 줄 정의
> **"카톡으로 흘려보내던 '거기 가고싶다'를 1초 만에 저장하고, 필요할 때 AI가 코스로 엮어주는 커플 위시리스트."**

### 1.2 포지셔닝 원칙 (핵심)
- **카톡을 대체하지 않는다.** 커플의 일상 대화는 여전히 카카오톡에서 일어난다.
- DateMate는 **"카톡에서 발견한 것을 저장하는 외부 인박스"** 역할. 카톡 공유 시트, 복붙, 링크 파싱이 핵심 진입점.
- AI 채팅 UI는 메인이 아니라 **"AI에게 물어보기" 서브 기능**으로 격하. 평소엔 조용한 저장소, 필요할 때만 추천.
- **솔로 사용도 가치 있게.** 커플 연동은 선택 사항. 혼자서도 위시리스트 앱으로 유용해야 함.

### 1.3 핵심 가치 제안 (Value Proposition)
- **제로 마찰 적재 (Zero-Friction Capture):** 카톡 공유 시트 1탭, 혹은 복붙 1초로 저장. AI가 링크·텍스트에서 장소·카테고리 자동 추출.
- **개인화 (Personalization):** 우리 커플이 직접 쌓은 위시리스트·기념일·선호도를 1순위 데이터로 활용.
- **커뮤니티 (Community):** 다른 커플들이 검증한 실제 데이트 장소 DB ("거지맵" 스타일) 활용.
- **요청형 AI (On-Demand AI):** "오늘 뭐할까?" 버튼을 눌렀을 때만 3단계 RAG로 코스 제안. 평소엔 방해하지 않음.
- **카톡으로 되돌리기 (Boomerang):** 저장한 코스/장소를 카톡 공유 포맷으로 1탭 내보내기. 의사결정은 여전히 카톡에서.

### 1.4 목표 규모 (Scale Target)
| 지표 | 목표 |
| --- | --- |
| 동시 접속자 (DAU Peak) | 500명 |
| 누적 가입자 | 10,000명+ |
| 월간 적재 건수 | 150,000건 |
| 월간 AI 추천 요청 수 | 50,000건 |
| 목표 MAU | 3,000명 |

## 2. 배경 및 문제 정의 (Problem Statement)

### 2.1 사용자 페인 포인트
1. **카톡 휘발성:** 연인과 카톡으로 "여기 가보고 싶다"며 링크·스샷 주고받지만, 며칠 지나면 스크롤 위로 올라가 사라짐. 막상 데이트 당일엔 못 찾음.
2. **저장 장소 분산:** 인스타 저장, 네이버 지도 즐겨찾기, 카톡 나에게 보내기 — 흩어져 있어 결국 못 씀.
3. **"오늘 뭐 먹지?" 무한루프:** 커플의 약 70%가 매주 겪는 의사결정 피로.
4. **추천 앱의 몰개성:** 기존 맛집 앱은 '우리 커플'의 예산·취향·기념일 맥락을 모름.
5. **블로그 광고 피로:** SEO 상위권 블로그의 과장된 리뷰, 협찬 피로.

### 2.2 기존 대체재 분석
| 서비스 | 한계점 |
| --- | --- |
| 카톡 "나에게 보내기" | 검색·구조화 불가, 카테고리·지도 뷰 없음 |
| 네이버 지도 즐겨찾기 | 커플 공유 불편, 텍스트 메모 한정, 상품·선물 저장 불가 |
| 인스타 저장 | 장소 정보 구조화 X, 지도 연동 X |
| 캐치테이블, 망고플레이트 | 맛집 한정, 저장·적재 워크플로 아님 |
| ChatGPT 등 범용 AI | 우리 커플 맥락 모름, 매번 재입력 |
| 데이트팝, 코코아북 | UGC 기반이나 개인 적재 기능 약함 |

**차별점:** **카톡 공유 시트에서 1탭 적재** + 커플 공유 + AI 링크 파싱 + 요청형 코스 추천 4중 결합. 경쟁자가 "추천 엔진"에 집중할 때, 우리는 "저장 워크플로"를 장악.

## 3. 타겟 사용자 (Target Persona)

### 3.1 Primary Persona — "계획형 여친 지영 (27세)"
- 연애 1년차, 매주 토요일 데이트 계획 담당.
- 카톡으로 남친에게 블로그·인스타 링크 자주 보냄. 그런데 막상 주말엔 그 링크를 못 찾음.
- **니즈:** "카톡에서 본 거 그 자리에서 바로 저장하고, 주말에 지도로 한눈에 보고 싶어."

### 3.2 Secondary Persona — "결정장애 남친 민호 (29세)"
- 계획 세우기 귀찮아함. "아무거나" 화법 자주 사용.
- 여친이 보낸 링크 저장 안 함. 주말에 여친이 "그때 보낸 거 가자" 하면 난감.
- **니즈:** "여친이 보낸 장소가 자동으로 어딘가 쌓여 있고, 주말에 AI가 골라줬으면."

### 3.3 Non-Target
- 미성년자 (14세 이하).
- 데이팅 앱 사용자 (우리는 **기존 커플** 대상, 매칭 서비스 아님).

## 4. 범위 (Scope)

### 4.1 In-Scope (MVP, v1.0)
- Google OAuth 로그인 및 커플 연동 (초대 코드 방식, 솔로 사용 허용)
- **빠른 적재 인박스:** 복붙 1초 등록, AI 링크·텍스트 파싱 → 카테고리·장소 자동 추출
- **PWA 공유 시트 타겟:** 카톡·브라우저에서 "공유 → DateMate"로 1탭 저장
- **카톡 내보내기:** 위시리스트/코스를 카톡 공유 포맷으로 클립보드·공유 시트로 송출
- 위시리스트 관리 (리스트·지도 뷰, 카테고리, 완료 처리, `added_by` 표시)
- **요청형 AI 추천:** "오늘 뭐할까" 버튼 → 3단계 RAG (Internal → Community → Web)
- 서브 기능으로서 AI 채팅 (사이드 탭, 수시 대화 아님)
- 공용 데이트 맵 (비로그인 열람 가능, SEO용)
- Admin 대시보드 (프롬프트 튜닝, 통계, 웹검색 토글)
- 이별 모드 (Unlink) 및 RLS 기반 데이터 격리
- 쿠팡 파트너스 딥링크 자동 삽입

### 4.2 Out-of-Scope (v1.0 제외, v2.0 이후)
- **커플 간 상시 대화/메신저 기능** (카톡이 이미 담당)
- **`Messages` 테이블·대화 로그 영속화** (프라이버시·이별 분쟁 리스크 제거)
- 네이티브 앱 (iOS/Android — PWA로 먼저 검증)
- 결제·유료 구독 (MVP는 애드센스+제휴링크만)
- 알림 서비스 (Push, Email Digest)
- 커플이 아닌 그룹 기능 (친구, 가족)
- 다국어 (초기 한국어 Only)

### F2. 빠른 적재 인박스 (Capture Inbox) — **메인 화면**
| 항목 | 내용 |
| --- | --- |
| F2.1 | 홈 화면 상단에 거대한 입력창 하나 + "붙여넣기" 힌트 ("카톡에서 본 거 그냥 붙여넣어 보세요") |

| F2.3 | URL 감지 시 **도메인별 어댑터(F2-A)** 경유 후 동일 파이프라인으로 합류 |
| F2.4 | 추출 결과를 **카드 Preview**로 보여주고 유저가 1탭 "저장" → Wishlist Insert |

| F2.7 | 적재 성공 시 파트너에게 실시간 반영 (Supabase Realtime) |

### F2-A. 멀티소스 링크 어댑터 (Link Adapters)
공통 인터페이스 `resolve(url) → NormalizedPayload`. 어댑터가 실패하면 범용 OG 스크랩 fallback.

| 소스 | URL 패턴 | 추출 전략 | 비용 |
| --- | --- | --- | --- |
| **카카오맵** | `kko.to/*`, `place.map.kakao.com/*`, `map.kakao.com/*` | 단축 URL 리다이렉트 추적 → `place_id` 파싱 → **카카오 로컬 API**로 상세(이름·주소·좌표·카테고리) | 무료 (일 10만) |
| **네이버 지도** | `naver.me/*`, `map.naver.com/*`, `m.place.naver.com/*` | 리다이렉트 추적 → 장소명 추출 → **네이버 로컬 검색 API**로 표준화 | 무료 (일 2.5만) |
| **구글 지도** | `maps.app.goo.gl/*`, `google.com/maps/*` | 리다이렉트 → OG `og:title`·좌표 파싱 → **카카오 로컬로 재검색**해서 국내 좌표계 정규화 | 무료 |
| **네이버 블로그** | `blog.naver.com/*`, `m.blog.naver.com/*` | iframe 내부 본문 스크랩 → Gemini Flash에 HTML 일부 주입 → `{title, address, category, summary, images[]}` JSON 강제 출력 | Gemini만 (≈0.2원/건) |
| **유튜브** | `youtube.com/*`, `youtu.be/*`, Shorts | **oEmbed 공식 API**로 제목·썸네일·채널 → 설명란 텍스트를 Gemini에 전달해 장소 정보 추출 (없으면 `category='영상참고'`로 저장) | 무료 |
| **인스타그램** | `instagram.com/p/*`, `instagram.com/reel/*` | OG 메타만 확보 가능 (API 제약). 위치 태그가 캡션에 있으면 Gemini 추출. 실패 시 "장소명을 알려주세요" 유저 프롬프트 → 카카오 로컬 보정 | 무료 |
| **일반 블로그/기사** | 그 외 모든 URL | OG 메타 + `<article>` 본문 1,500자 → Gemini 파싱 | Gemini만 |

**공통 파이프라인:**
```
URL 수신
 → 1) Adapter 선택 (도메인 매칭)
 → 2) 1차 소스 획득 (공식 API / oEmbed / OG)
 → 3) 필요 시 HTML 본문 스크랩 (5초 타임아웃)
 → 4) Gemini Flash로 { title, category, address, tags, summary } JSON 추출
 → 5) 주소 → 좌표 정규화 (카카오 Geocoding)
 → 6) 썸네일 URL 검증 (HEAD 요청)
 → 카드 Preview 반환
```

**비용 가드:**
- 적재 1건당 Gemini Flash 사용량 ≈ 500 input + 300 output 토큰 → **약 0.2원/건**
- 월 150,000 적재 가정 시 Gemini 비용 ≈ **$20 이하**
- 외부 API는 무료 한도 내. 초과 시 `429` → 큐잉 후 다음 날 재시도
- 동일 URL 24시간 내 재요청은 캐시 히트(`Capture_Cache` 테이블), Gemini 호출 생략

**규정 준수:**
- `robots.txt` 존중, 요청당 User-Agent 명시(`DateMateBot/1.0 (+https://datemate.kr/bot)`)
- 유저가 "공유"한 URL만 처리 (크롤러 아님 — 유저 트리거형 단건 fetch)
- 인스타/네이버 블로그는 본문 저장 없이 **추출된 구조화 필드만 영속화** (저작권 리스크 축소)

**안정성:**
- HTML 구조 변경으로 파서가 깨질 때를 대비해, 어댑터는 "원문 HTML → Gemini JSON 강제 출력" 전략 우선. 정규식·셀렉터 의존도 최소화.
- 어댑터 실패율을 `Capture_Logs`에 기록 → Admin 대시보드에서 도메인별 성공률 모니터링

### F3. 카톡 연동 (Bidirectional)
| 항목 | 내용 |
| --- | --- |
| F3.1 | **공유 시트 진입(Inbound):** PWA `manifest.webmanifest`의 `share_target` API로 카톡 "공유" 리스트에 DateMate 노출 → 선택 시 F2 파이프라인 자동 실행 |
| F3.2 | **복붙 진입(Inbound):** 홈 입력창에 텍스트/링크 붙여넣으면 즉시 F2 파이프라인 |
| F3.3 | **카톡 내보내기(Outbound):** 위시리스트 아이템·추천 코스 옆 "📤 카톡 공유" 버튼 → Web Share API 호출. 미지원 환경은 클립보드 복사 fallback |
| F3.4 | 내보내기 포맷: 제목 + 한 줄 요약 + 지도 링크(카카오맵 URL) — 카톡에서 썸네일 프리뷰 뜨도록 최적화 |
| F3.5 | 코스 전체 공유: "토요일 데이트 코스 3곳" 형태의 텍스트 템플릿 + 개별 지도 링크 리스트 |

### F3-A. AI에게 물어보기 (서브 기능)
| 항목 | 내용 |
| --- | --- |
| F3A.1 | 메인 동선이 아님 — 하단 탭 또는 홈 카드 중 하나로 배치 |
| F3A.2 | "오늘 뭐할까?" 원클릭 CTA → 현재 시각·날씨·파트너 위치 컨텍스트로 3단계 RAG 실행 |
| F3A.3 | 자유 질의 입력도 허용하되 **대화 히스토리 미저장** (세션 한정) |
| F3A.4 | 가드레일: 데이트 외 주제는 "저는 데이트만 도와드려요 😊"로 응답 |
| F3A.5 | 스트리밍 응답 (SSE) 지원, 추천 결과에 "위시리스트 추가" 버튼 |

### F3-B. 3단계 RAG 추천 로직 (F3-A의 내부 동작)
```
유저 요청 "오늘 뭐할까?" 또는 "이번 주말에 성수동 갈까?"
 ↓

   → hits >= 2 이면 Step 1 결과로 답변 구성, 종료
 ↓
[Step 2] Public_Places 검색 (성수동 + rating desc)
   → hits >= 3 이면 Step 1 + Step 2 결과 합성
 ↓
[Step 3] Web Search (Admin 토글 ON 시에만)
   → Gemini Google Search Grounding 호출
 ↓
최종 답변 생성 (소스 플래그는 내부 메타데이터로만 기록, 유저에게 노출 X)
```

## 6. 비기능 요구사항 (Non-Functional)

### 6.1 성능
- AI 채팅 첫 토큰 응답: **< 2초** (Gemini Flash for Intent)
- 전체 스트리밍 완료: < 10초 (Pro 기준)
- 페이지 초기 로딩 (LCP): < 2.5초

### 6.2 보안
- **RLS:** 모든 커플 데이터 테이블에 `couple_id = auth.jwt()->couple_id` 정책 필수
- **Prompt Injection 방어:** 유저 입력을 시스템 프롬프트 바깥 `<user_input>` 태그로 래핑
- **API Key 격리:** Gemini 키는 서버 전용 (Next.js Server Action / Route Handler)
- **Rate Limit:** 유저당 분당 20 메시지 (Upstash Redis)

### 6.3 프라이버시
- 이별 모드: `Couples.status = 'broken'` 전환 시 30일 후 커플 데이터 자동 삭제 (크론)
- 유저는 자신의 데이터만 Export/Delete 요청 가능 (GDPR/PIPA 대응)

### 6.4 확장성
- Supabase Free Tier → Pro 전환 기준: MAU 500 or DB 500MB 초과
- Vercel Edge Function 활용으로 지역 레이턴시 최적화

## 7. 데이터 모델 요약 (상세 SQL은 별도 문서)

```
Users ─┐
       ├── Couples (user1_id, user2_id)
       │      └── Wishlist (couple_id FK)   ← 유일한 커플 데이터
       │                └── source_url, source_type, thumbnail 필드 포함
       │
       └── Public_Places (기여자 user_id)

System_Configs ── Admin 전용
API_Logs       ── 비용/사용량 추적
Capture_Logs   ── 적재 파이프라인 이벤트 (어댑터별 성공률, 품질 개선)
Capture_Cache  ── URL → 파싱 결과 24h 캐시 (Gemini 중복 호출 절감)
```
**변경점:** `Messages` 테이블 제거 — 일상 대화는 카톡에 남기고, DateMate는 위시리스트만 영속화. 이별 모드 구현·프라이버시·DB 비용 모두 감소.

## 8. 사용자 플로우 (Core User Journey)

### 8.1 신규 가입 → 첫 적재 (솔로도 가능)
```
Google 로그인 → 온보딩 (닉네임) →
"나중에 파트너 초대하기" 건너뛰기 허용 →
홈 화면 입력창 "카톡에서 본 거 붙여넣어 보세요" →
유저가 블로그 URL 붙여넣기 →
AI가 카드 Preview 생성 ({제목, 카테고리:카페, 주소:성수동}) →
유저 "저장" 탭 → Wishlist 첫 항목 생성 →
"파트너도 초대하면 같이 볼 수 있어요" 배너 노출
```

### 8.2 카톡 공유 시트 적재 (킬러 플로우)
```
카톡 대화방: 친구가 블로그 링크 전송
 ↓ 유저가 링크 길게 누르기 → "공유" → DateMate 선택
 ↓ (PWA share_target이 URL 수신)
DateMate가 백그라운드에서 메타 스크랩 + Gemini 파싱
 ↓ 카드 Preview 2초 내 표시
유저 "저장" 1탭 → Wishlist 등록 → 카톡으로 자동 복귀
파트너: Supabase Realtime으로 즉시 알림 없이 조용히 반영
```

### 8.3 주말 활용 플로우 (요청형)
```
토요일 오후, 유저가 DateMate 열기 →
홈 상단 "오늘 뭐할까?" 버튼 탭 →
3단계 RAG 실행 → 위시리스트에서 2곳 + 커뮤니티 1곳 조합한 코스 제안 →
유저 "좋아" → "📤 카톡 공유" 탭 →
카톡 파트너에게 "토요일 코스: ①성수 카페 ②한강 산책 ③압구정 와인바 (지도 링크 3개)" 전송 →
의사결정과 당일 대화는 카톡에서 계속됨
```
